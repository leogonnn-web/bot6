# Гидра — Математический анализ оптимальных настроек

## Архитектура стратегии

Гидра = **Мартингейл-сетка (3 уровня)** + **Knife-catch (вход на откате)**

```
Entry: цена упала ≥ drop_threshold (0.03%) → покупаем
Grid L2: цена упала ещё 0.4% → покупаем 1.5x
Grid L3: цена упала ещё 0.4% → покупаем 1.5x
TP: +0.2% от средней цены сетки
SL: panic_stop 2% от entry (или breakeven timeout 20 мин)
```

## Математика Мартингейла

Пример: entry_price = $1.00, slot = $5

| Уровень | Цена | USDT | Монет | Средняя | TP 0.2% | До TP |
|---------|------|------|-------|---------|---------|-------|
| L1 only | $1.000 | $5.00 | 5.000 | $1.000 | $1.002 | +0.20% |
| L1+L2 | $0.996 | $12.50 | 12.563 | $0.997 | $0.999 | +0.28% |
| L1+L2+L3 | $0.992 | $23.75 | 23.967 | $0.991 | $0.993 | +0.38% |

**Вывод**: чем глубже сетка заполняется, тем выше нужен отскок для TP.
- L1 only: отскок 0.20% от entry
- L3: отскок 0.38% от entry (но уже упали на 0.80%)

## Ключевое уравнение: Expected Value (EV)

```
EV = (WinRate × AvgWin) - (LossRate × AvgLoss)
```

### Режим 1: Без анализатора (текущий после наших изменений)

- **Частота входов**: ~1 сделка/3-5 мин на 11 символах (по логам)
- **WinRate на L1**: ~80-90% (отскок 0.2% после 0.03% отката на шиткоинах)
- **WinRate на L3**: ~60-70% (отскок 0.38% после 0.80% просадки)
- **LossRate**: ~10-20% (если начинается тренд вниз)
- **AvgWin**: $0.01 (0.2% от $5)
- **AvgLoss**: $0.10-$0.47 (panic_stop 2% от $5 = $0.10, но если L3+SL = $0.47)

**EV L1**: 0.85 × $0.01 - 0.15 × $0.10 = $0.0085 - $0.015 = **-$0.0065** (отрицательный!)
**EV L3**: 0.65 × $0.01 - 0.35 × $0.47 = $0.0065 - $0.1645 = **-$0.158** (очень отрицательный!)

**Проблема**: TP слишком мал (0.2%), SL слишком велик (2%). Нужно соотношение TP:SL ≥ 1:1, иначе нужен WinRate > 90%.

### Режим 2: С анализатором (текущий, строгий)

- **Частота входов**: ~1 сделка/30-60 мин
- **WinRate**: ~70-80% (лучшие точки входа)
- **AvgWin**: тот же $0.01
- **AvgLoss**: тот же $0.10-$0.47

**EV**: 0.75 × $0.01 - 0.25 × $0.10 = $0.0075 - $0.025 = **-$0.0175**

Слишком мало сделок, EV всё равно отрицательный из-за дисбаланса TP/SL.

## Корневой вывод

**Анализатор не виноват. Проблема в соотношении Risk/Reward.**

TP 0.2% / SL 2% = 1:10. Для прибыльности нужен WinRate > 90.9%.
Это нереалистично даже с идеальным анализатором.

## Решение: изменить Risk/Reward

Вариант А: **Увеличить TP до 1.5%** (как в оригинальном config)
- TP 1.5% / SL 2% = 1:1.33
- Нужный WinRate: >57% — реально с анализатором
- Но: сетка будет висеть дольше, шиткоины колеблются

Вариант Б: **Снизить SL до 0.8%** (trailing stop на гриде)
- TP 0.2% / SL 0.8% = 1:4
- Нужный WinRate: >80% — реально без анализатора на шиткоинах
- Быстрые сделки, много входов

Вариант В: **Динамический TP/SL по ATR**
- TP = ATR × 0.5
- SL = ATR × 2.0
- Адаптируется под волатильность символа

## Рекомендация для Гидры (накопитель)

**Цель**: максимум сделок с положительным EV.

```
drop_threshold: 0.15%    (было 0.03% — слишком частые ложные входы)
hydra_net.take_profit_pct: 0.8%  (было 0.2% — убытков больше, чем прибыли)
hydra_net.grid_distance_pct: 0.6%  (было 0.4% — реже, но глубже)
panic_stop: 1.2%  (было 2.0%)
breakeven_timeout_sec: 600  (10 мин вместо 20 — не ждём вечно)
```

**Анализатор**: НУЖЕН, но в **скальп-режиме**:
- min_confidence: 30% (было 60%)
- Игнорировать EMA alignment (knife-catch = покупаем ПРИ падении)
- RSI oversold: < 45 (было < 30)
- MACD: игнорировать bearish, проверять только histogram reversal
- Без Ichimoku блокировок

## Проверка: EV с новыми параметрами

- TP 0.8% / SL 1.2% = 1:1.5
- Нужный WinRate: >65%
- На шиткоинах с анализатором (ослабленным): ~70-75%
- Сделок: ~1/5-10 мин
- EV: 0.72 × $0.04 - 0.28 × $0.06 = $0.0288 - $0.0168 = **+$0.012** ✅
